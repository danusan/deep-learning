“Deep Learning for Coders” (Fragen 1-9)

  1. Can we always use a random sample for a validation set? Why or why not? random sampling is a perfectly fine way to create a validation set. The training set and test set come from roughly the same distribution so your randomly sampled validation set is a good proxy for the test set. Typically when your model improves on your validation set, you'll also see improvements on your test set.
  2. What is overfitting? Provide an example. (With training set and validation set measure accuracy) If you train for too long, with not enough data, you will see the accuracy of you model start to get worse; this is called overfitting Example: If our model does much better on the training set than on the test set, then we're likely overfitting. For example, it would be a big red flag if our model saw 99% accuracy on the training set but only 55% accuracy on the test set.
  3. What is a metric? How does it differ from "loss"? A metric is used to judge the performance of your model. This is only for you to look at and has nothing to do with the optimization process. (What I care about=> error rate)

The loss function is used to optimize your model. This is the function that will get minimized by the optimizer. (computer is using it=>measure of performance)
